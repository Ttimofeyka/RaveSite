Variables

I think everyone who has studied programming or mathematics a little knows what variables are.
In Rave, they work almost like in C, but there are still differences.
You will learn about these differences here.

1. In Rave, all variables are automatically initialized to null.

In programming languages like Zig, it is required to explicitly indicate that a variable is uninitialized, because uninitialized variables can cause undefined behavior.

In the Rave, we solved this problem by automatically initializing all variables.
Of course, you can disable automatic initialization if necessary.

Example: 
        int a;
long b;
All these variables are not initialized by the user, so the compiler automatically initializes them with the value 0.
Example where automatic initialization is explicitly disabled: 
    (nozeroinit) int c;
Did you notice that we used variable options?
The next paragraph is about them.

2. There are options for functions and structures in the Rave

In Rave, all options for functions and variables are in parentheses and are in front of before their declarations.
There are much fewer options for variables than for functions, but they still exist - "nozeroinit" and "volatile".

We discussed the "nozeroinit" option in the previous paragraph - it prohibits the compiler from automatically initializing a variable.

The "volatile" option marks the variable as volatile.
The purposes of this option are identical to the same option in the C programming language.
Example: 
        (volatile, nozeroinit) short d;
(volatile) cent e = 0;
